# C++ Coding Standards


Variáveis:

Nome de variáveis:
Variaveis locais devem possuir um nome significativo seguindo o padrão camelCase
começando sempre com a primeira letra em minuscula

int windowWidth = 700;

Variáveis para loops devem possuir uma letra e seguir o padrão matematica i, j, k, l...

Deve-se evitar utilizar variáveis globais, se for necessário defini-las como constantes
e seu nome deve ser todo em maiusculo e contendo undercores para separar cada palavra.

const int MAX_TIME_OPEN = 400;

Para utilizar variáveis estaticas deve serguir a mesma regra do nome de variáveis
static bool continueOpen = true;

Deve-se evitar utilizar abreviações o máximo possível, porém caso o nome da variável fique muito longo
recomenda-se que utilize abreviações em palavras comumente conhecidas
Application currentRunningApp;

Variáveis do tipo bolean devem possuir uma pergunta se possível:
bool isActive
bool hasErrorCode
bool shouldClose


Namespace:

Deve-se utilizar sempre um namespace se possível, quando não for possível utilizar um namespace
deve-se utilizar nome de funções e variáveis facílmente reconheciveis como sendo
da sua biblioteca:

void DATFileOpenFile(const std::string& fileName);

const int DATFILE_MAX_TIME_OPEN = 400;

Deve-se evitar utilizar using namespace em namespace's diferentes do seu domínio,
como no std, caso utilize com frequencias alguma função ou membro de um namespace
deve-se optar por utilizar using std::endl;

RUIM: using namespace std;
BOM: using std::endl;


Classes

O nome de uma classe deve ser camelCase porém com a primeira letra em maiusculo,
o nome dos arquivos .h e .cpp deve serguir a mesma lógica.

class MyClass {
...
}


Membros das classes devem possuir o nome prefixado com um m minusculo:

private:
	bool mAccountNumber;


Métodos devem possuir o nome em camelCase e evitar abreviações
void printMessage();

Métodos que retornam valor deve possuir o prefixo get, métodos que definem valor(es)
devem possuir o prefixo set

int getNumberOfLines();
void setNumberOfLines(int lines);

Deve-se sempre informar o nome dos argumentos, para IDE's auxiliarem na hora do type hint
